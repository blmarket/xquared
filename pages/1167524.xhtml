Building

This document explains how to create Xquared's distribution version from the source code.

 

Receiving the source code from storage

Xquared uses GoogleCode(SVN) as a repository. If you are new to SVN, please read this article and enter the following command to get the source code:

 

  1. svn checkout http://xquared.googlecode.com/svn/trunk/ xquared-read-only

 

Installing necessary libraries/programs for building: Installing Ant

The Xquared build process has been written through Apachi Ant ( http://ant.apache.org/ ). Thus, you must go to http://ant.apache.org/  and download the latest distribution version and install it.

 

 

Installing JSDoc_toolkit

in Xquared, JSDoc_toolkit is used to automatically generate API documents from the JavaScript source. You must install the latest release version from http://code.google.com/p/jsdoc-toolkit/

 

 

Building

The complete build process is as follows:

  1. Restoring the distribution directory (dist)

  2. Merging(utlize Ant's tar task) JavaScript files in order to generate xquared.js

  3. Compress merged files and delete comments to generate xquared-min.js
  4. Extract comments from files that have not been merged to generate an API document (use jsdoc_toolkit)
  5. Compress all content in the distribution directory to generate xquared.tar.gz

 

To execute the build, first open build.properties and after  properly edit the jsdoc_home path input

 

  1. > ant

 

to automatically start building. When building has been completed, the following directories will be generated:

  • dist/doc/api - automaitcally generated API documents
  • dist/css - provided CSS files by default
  • dist/js - xquared.js (includes all scripts into one) and xquared-min.js (compressed version)
  • dist/examples - example files that can be executed locally
  • README, LICENSE
  • xquared.tar.gz - compressed files for redistribution. This file includes all files in the dist directory.
