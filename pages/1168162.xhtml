Create and look over specifications

One of the main goals in Xquared is to have harmonious execution and functionality in all browsers. Automatic tests (or specs) are very important. Xquared makes use of JSSpec and it executes automated tests. Currently there are hundrends of different automated test situations that have been created.

 

Therefore, you don't have to be worried about something that may have went wrong during code editing. This is because If you open the HTML files inside the Xquared source code directory, an automated test will be executed to see if there are any problems.


  1. /xquared/specs/specs/


Using the tab browsing feature in your browser will allow simultaneous execution of all specs. Also, bookmarking all open tags will allow you to run tests more easier in the future.

 



[A sccrenshot when running all specs on Firefox 2.0]


Before you start editing and fixing bugs, first you should get into the habit of adding the "will fail if the corresponding bug is not fixed" test and start working on editing the bug. By doing this, you can easily detect errors of the corresponding bug that was fixed through automated tests. Please refer to the JSSpec User Guide on basic test creation.

 

Finished editing?

Now it's time to create a patch. Before creating patches, do not forget to receive and combine the latest code from Xquared's homepage. From where you have stored your files use

  1. hg pull


to bring the most recent source, then use


  1. hg update


to renew your storage area. Next use


  1. hg export > patch.diff


to create a patch file.  If you receive a confliction problem after you hg update please refer to the merging confliction modification manual.


Upload it to ECUS

ECUS is the place where you and the Xquared team communicates. Everything that you upload here is public and shared so that your contributions are progressed.


First, log in to http://ecus.openmaru.com. After that, select Xquared from the project menu on the left side, and select 'Write'. On the title field, input [PATCH], and in the content field write a description on the purpose of patch and what it does. Attach the patch and then upload. The below is a simple example.


Title: [PATCH] It's not 'webkit', but it's 'WebKit'.

Content: I know it's something small, but I thought it needed to be corrected.


Reflection

The patch you have submitted will be received and reviewed by a member of Xquared. We will notify you on the status of how it is being reflected. If it becomes reflected, then you will see a blue colored "Resolved" mark.


Now you have contributed your work! Everyone that has made contributions is greatly appreciated and will be recognized in our Contribution Temple.
