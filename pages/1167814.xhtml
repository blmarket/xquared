Set Up

You can easily modify the Xquared’s functionality by appointing set points:

 

  1. xed.config.enableLinkClick = true;

 

Please refer to the table below on what set points can be utilized (in alphabetical order):

Name

Value

Note

autoFocusOnInit

false

Focus is given automatically when the editing mode is executed if ‘true’ is set.

automaticallyHookSubmitEvent

true

The element connected to Xquared is the text area, and when set to ‘true’ before a form is submitted the value of the text area is automatically renewed when the corresponding text area is put into a form.

bodyId

''

Applied ID value in the body tag of the WYSIWYG editor.

bodyClass

'xed'

Applied class value in the body tag of the WYSIWYG editor.

changeCursorOnLink

false

When in editing mode, putting the mouse over a link will change to a hand cursor when set to true. To enable the link to be clicked you must change the enableLinkClick settings.

contentCssList

['css/xq_contents.css']

List of CSS used in WYSIWYG mode. Through the .Editor.loadStylesheet() function you can dynamically add CSS. However, if you want to appoint the CSS during initial loading through the setting value, you may experience screen flickering.

defaultToolbarButtonMap

[...]

Arrangement of buttons that go into Xquared’s default toolbar. You can change the arrangement, categories, and location of buttons by changing this value. For more detailed information please refer to User defined toolbar.

enableLinkClick

false

When ‘true’, clicking the link would take you to the corresponding URL. You  have to change the settings on changeCursorOnLink if you want the cursor to change to hand icon when hovering over a link with the mouse.

generateDefaultToolbar

true

Sets up whether or not you will show Xquared’s provided default toolbar. Set this to ‘false’ to utilize a user defined toolbar.

imagePathForContent

'img/content/'

Path of utilized images within the editor

imagePathForDefaultToolbar

'img/toolbar/'

Path for toolbar buttons, background, dividers, and images.

noValidationInSourceEditMode

false

Validation is not attained during the calling of getCurrentContent() or the submission of a form when set to ‘true’. However, validation is attained with no relation to the option of converting from the source editor mode to WYSIWYG mode.

urlValidationMode

'absolute'

Sets up the validation method for the URL. For example, currently in this type of URL "http://a.com/b/c.html?d=1" a link inside a document (i.e., href) is converted depending on the mode (shown below).

'relative' (converted to the shortest relative path):

·         http://a.com/b/c.html?d=1 --> #

·         c.html?d=2 --> ?d=2

·         http://a.com/b/d.html --> d.html

·         http://x.com/b.html --> http://x.com/b.html

 

'host_relative' (‘absolute’ if the host is different, ‘path’ if the host is identical parts that come after ‘path’ will be preserved’.):

·         http://a.com/b/c.html?d=1 --> /b/c.html?d=1

·         c.html?d=2 --> /b/c.html?d=2

·         http://a.com/b/d.html --> /b/d.html

·         http://x.com/b.html --> http://x.com/b.html

 

'absolute' (coverts to an absolute path):

·         http://a.com/b/c.html?d=1 --> http://a.com/b/c.html?d=1

·         c.html?d=2 --> http://a.com/b/c.html?d=2

·         http://a.com/b/d.html --> http://a.com/b/d.html

·         http://x.com/b.html --> http://x.com/b.html

 

'browser_default' (Follows the default settings of the browser):

·         Different depending on the browser type.

widgetContainerPath

'widget_container.html'

Widget container HTML file path

whitelist

{...}

Object that has property lists classified by tags and tag lists approved by the Validator.

 

{ 'a': ['href', 'id', 'class', ...],
   'blockquote': ['id', 'class', 'cite', ...],
   ....
}
